## Project Overview

Company Information
Name: Exceed Education
Motto: We are more than a tutoring centre. We are a platform for growth. We are here to make learning exciting again.
Description: A tutoring company for high school students in NSW Australia.

This project is a website to showcase this company. It is a Gatsby 5 static site with React 19, TypeScript, and serverless functions. Uses Contentful as a headless CMS and Fibery as a member database backend. Deployed on Netlify.

## Commands

```bash
npm run develop       # Dev server at http://localhost:8000
npm run build         # Production build
npm run serve         # Serve production build locally
npm run clean         # Clear Gatsby cache (useful for stale build issues)
npm run typecheck     # TypeScript type checking (tsc --noEmit)
npm run lint          # ESLint (fails on errors; warnings are pre-existing any types)
npm run lint:fix      # ESLint with auto-fix
npm run format        # Prettier with Tailwind class sorting (writes files)
npm run format:check  # Prettier check only (CI-safe)
npm run test          # Vitest single-pass run
npm run test:watch    # Vitest interactive watch mode
npm run test:coverage # Vitest with V8 coverage report in ./coverage/
```

## Architecture

**Stack:** Gatsby 5 + React 19 + TypeScript (strict mode) + Tailwind CSS 4 + daisyUI 5

**Data flow:**

1. **Build time:** Gatsby queries Contentful CMS via GraphQL (`gatsby-source-contentful`) to generate static pages
2. **Runtime forms:** Client-side Formik forms with Yup validation schemas (shared between client and server in `src/utils/schemas.ts`)
3. **API calls:** Gatsby Functions in `src/api/` become Netlify serverless functions — currently `create-member.ts` POSTs to Fibery API to create members in `Space/Members` database

**Key directories:**

- `src/pages/` — Gatsby file-based routing (index, about-us, contact-us, timetable, 404)
- `src/components/` — React components (Header, MemberForm, TutorCard, TutorsSection)
- `src/api/` — Serverless function endpoints (mapped to `/api/*` routes)
- `src/utils/` — Shared validation schemas (Yup)
- `src/styles/global.css` — Tailwind + daisyUI imports

**Contentful integration:** Landing page queries use hardcoded Contentful entry IDs (e.g., `"4QyBkGkABsUBblJqxOZCov"`) to fetch `contentfulContentCard` entities. Tutor data uses rich text fields rendered via `@contentful/rich-text-react-renderer`.

**Styling:** All styling is utility-class based via Tailwind CSS + daisyUI components. No CSS modules or CSS-in-JS. Primary brand color is `#e95a00` (orange). daisyUI provides light/dark theme support.

## Environment Variables

Copy `env.template` to `.env.development` (and `.env.production` for builds):

- `CONTENTFUL_ACCESS_TOKEN` / `CONTENTFUL_SPACE_ID` — Contentful CMS access
- `FIBERY_TOKEN` / `FIBERY_WORKSPACE` — Fibery API for member creation

## Conventions

- All source files are TypeScript (`.tsx` for components/pages, `.ts` for API/utils)
- Form validation uses Yup schemas defined in `src/utils/schemas.ts`, shared between MemberForm component and `/api/create-member` endpoint
- API endpoints return standardized `ErrorResponse` type: `{ error: string, details?: string, message?: string }`
- GraphQL types are auto-generated by Gatsby (see `src/gatsby-types.d.ts`)
- Gatsby config loads env vars via `dotenv` based on `NODE_ENV`

## Tooling

**ESLint 8** (`.eslintrc.cjs`) — TypeScript, React, hooks, and jsx-a11y rules. Uses ESLint 8 (not 9) because Gatsby 5 bundles `eslint-webpack-plugin` with a peer dep of `eslint: "^7 || ^8"`.

**Prettier 3** (`.prettierrc`) — includes `prettier-plugin-tailwindcss` for automatic Tailwind class sorting.

**Vitest + React Testing Library** (`vitest.config.ts`, `tsconfig.test.json`) — test files live alongside source as `*.test.ts` / `*.test.tsx`. Currently covers:
- `src/utils/schemas.test.ts` — Yup schema validation rules
- `src/utils/validation.test.ts` — validateMemberForm / validateWithSchema helpers
- `src/components/MemberForm.test.tsx` — form render, Formik validation, success/error flows
- `src/api/create-member.test.ts` — API handler method guard, validation, Fibery integration

When testing components that use Formik, always use `await waitFor(...)` — Formik validation is async. Mock `fetch` via `vi.stubGlobal("fetch", vi.fn())` and restore with `vi.restoreAllMocks()` in `afterEach`.
